RAW FEATURES

KEY FEATURES

  • Target: salary_usd (annual salary in USD)
  • Categorical: job_title, experience_level (EN/MI/SE/EX), employment_type (FT/PT/CT/FL), company_location (50+ countries), company_size (S/M/L), education_required, industry

  • Temporal: posting_date, application_deadline
  • Text: required_skills (comma-separated list)

DATA PREPROCESSING

THE PROBLEM: MISSING VALUES

OLD APPROACH: Drop all rows with missing values → DATA LOSS

OUR SOLUTION: INTELLIGENT IMPUTATION

SALARY (TARGET)

Median imputation by job similarity

Groups by experience + employment type

Preserves data patterns

OTHER FEATURES

Job title → "Not Specified"

Skills → Empty list (num_skills=0)

Numeric → Median imputation

Categorical → Mode or "Unknown"

RESULT: ZERO ROWS DROPPED → More training data → Better predictions!

FEATURE ENGINEERING

CREATED FEATURES

  • Temporal Features: posting_year, posting_month, days_to_apply
  • Skill Analysis: num_skills (count of required skills)
  • Binary Skill Flags: has_python, has_aws, has_docker, has_kubernetes, has_sql, has_deep_learning, has_nlp, has_tensorflow, has_pytorch

WHY FEATURE ENGINEERING?

Transform raw data into meaningful patterns that machine learning models can learn from. Example: "Python, AWS, Docker" → has_python=1, has_aws=1, has_docker=1, num_skills=3

ENCODING

Categorical variables (text) → Numerical values using Label Encoding

experience_level: Entry → 0, Mid → 1, Senior → 2, Executive → 3

MODEL SELECTION: RANDOM FOREST

WHY RANDOM FOREST?

  • Handles mixed data types naturally
  • Captures non-linear salary patterns
  • Finds feature interactions automatically
  • Robust to outliers
  • No feature scaling needed
  • Provides feature importance rankings
  • Prevents overfitting via ensemble

FEATURE IMPORTANCE

Perfect Fit: Salary prediction has complex, non-linear patterns (Junior to Mid big jump, skill combinations, location effects). Random Forest handles all of this naturally.

MATHEMATICAL FORMULATION

Random Forest as an Ensemble Method

Prediction Function:

ŷ = (1/B) Σ(i=1 to B) f_i(x)

Where B = number of trees (100), f_i(x) = prediction of tree i

Loss Function (MSE)

L = (1/n) Σ(y - ŷ)²

Minimized during training

R² Score (Accuracy)

R² = 1 - (SS_res / SS_tot)

Measures prediction quality

Feature Importance

Importance(j) = Σ(t∈T) Δi(s,j) × p(t) / Σ(k=1 to K) Σ(t∈T) Δi(s,k) × p(t)

Calculated by decrease in impurity when splitting on feature j

IMPLEMENTATION PIPELINE

RANDOM FOREST PROCESS

1. Bootstrap Sampling: Create 100 random data subsets

2. Build Trees: Train 100 decision trees on different subsets

3. Random Features: Each split uses random feature subset

4. Ensemble Voting: Average all tree predictions
